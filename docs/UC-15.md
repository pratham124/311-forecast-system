# Use Case UC-15: Weather-Aware Forecasting and Storm-Mode Alerting

**Goal in Context**: Ensure forecasts account for weather context and alerts align with storm-mode operations, so operational managers receive reliable risk-aware guidance during abnormal conditions.  
**Scope**: Operations Analytics System  
**Level**: User Goal  
**Primary Actor**: Operational Manager  
**Secondary Actors**: Forecasting Engine, Surge Detection Module, Weather Data Service, Notification Service, Data Storage System  
**Trigger**: Weather-influenced forecast runs and surge/anomaly conditions requiring storm-mode-aware alert behavior.

---

## Clarification

For UC-15, **storm mode is the same operational concept as the confirmed surge/anomaly state in UC-11**. UC-15 does not define a second storm-mode mechanism.

---

## Success End Condition
* Forecast behavior reflects weather-aware context where supported.  
* Storm mode references align to the UC-11 surge/anomaly state.  
* Alert and notification outcomes remain operationally traceable.

## Failed End Condition
* Weather context cannot be used safely or notification delivery fails, but system behavior remains baseline-compatible and traceable for follow-up.

---

## Preconditions
* Weather context is available for supported forecasting scenarios, or baseline-compatible fallback behavior is available.  
* Surge/anomaly confirmation logic (UC-11) is available as the canonical storm-mode source.  
* Alerting and notification pathways are operational.

---

## Main Success Scenario
1. System processes forecast behavior with available weather context.
2. System identifies or references storm mode through the confirmed UC-11 surge/anomaly state.
3. System applies storm-mode-aware alert behavior for relevant scope.
4. System evaluates demand risk under the active operational context.
5. Notification Service sends alerts when alert conditions are met.
6. System records forecast behavior context, storm-mode state, alert outcome, and notification outcome for review.

---

## Extensions
* **1a**: Weather context unavailable or unusable  
  * 1a1: System records the weather-context issue.  
  * 1a2: System continues with baseline-compatible forecast behavior.

* **2a**: Surge/anomaly condition not confirmed  
  * 2a1: Storm mode remains inactive.  
  * 2a2: Alert behavior remains baseline.

* **5a**: Notification delivery failure  
  * 5a1: System records delivery failure.  
  * 5a2: Event is marked for retry or manual follow-up.

---

## Related Information
* **Priority**: High  
* **Frequency**: Ongoing during forecast operation and event-driven when surge/anomaly conditions occur  
* **Open Issues**: None for UC-15 definition; storm-mode vocabulary is explicitly aligned with UC-11.

# UC-15 Fully Dressed Scenario Narratives  
**Use Case:** Weather-Aware Forecasting and Storm-Mode Alerting

---

## Main Success Scenario Narrative

Forecast operation runs with weather-aware context for supported scenarios.

When abnormal demand conditions are confirmed through surge/anomaly logic, that shared state is treated as storm mode for UC-15.

Alert behavior follows the storm-mode-aware operational path for relevant scope. If alert conditions are met, notifications are sent.

Operational records preserve a coherent trail showing forecast behavior context, storm-mode state, alert evaluation, and notification outcome.

---

## Alternative Scenario 1a — Weather Context Unavailable

Weather context cannot be used for the run.

The system records the condition and continues with baseline-compatible forecast behavior.

Operational continuity is preserved.

---

## Alternative Scenario 2a — Storm Mode Inactive

A potential abnormal condition is not confirmed.

Storm mode remains inactive and alert behavior remains baseline.

The outcome remains traceable in operational records.

---

## Alternative Scenario 5a — Notification Delivery Failure

A valid alert is created, but delivery fails.

The system records the failure and marks the event for retry or manual follow-up.

The operational manager may not receive the alert immediately, but the follow-up path remains explicit.

---

## Key Behavioral Theme Across All Alternatives

Across all scenarios:

- Weather-aware forecasting behavior is preferred when supported  
- Baseline-compatible behavior is used safely when weather context cannot be used  
- Storm mode in UC-15 always refers to the UC-11 surge/anomaly state  
- Alert and notification outcomes remain traceable for operational review
